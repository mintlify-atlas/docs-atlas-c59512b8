## Changing Colors

<Steps>
  <Step title="Open Tailwind config">
    Navigate to `tailwind.config.mjs` in the root of your project.
  </Step>

  <Step title="Modify color values">
    Edit the hex color values in the `theme.extend.colors` object. For example, to change the primary accent from orange to blue:

    ```javascript
    accent: {
      500: "#0066FF",  // Changed from #FF5C00 to blue
      // Adjust other shades accordingly
    }
    ```
  </Step>

  <Step title="Test your changes">
    Run `bun dev` to see your color changes in real-time. The portfolio uses these colors throughout for backgrounds, text, borders, and accents.
  </Step>
</Steps>

## Creating a Custom Color Palette

To add a completely new color palette:

```javascript
// In tailwind.config.mjs
theme: {
  extend: {
    colors: {
      // Keep existing colors (gray, dark, accent)
      // ...
      
      // Add your custom palette
      brand: {
        50: "#E6F7FF",
        100: "#BAE7FF",
        200: "#91D5FF",
        300: "#69C0FF",
        400: "#40A9FF",
        500: "#1890FF",  // Base color
        600: "#096DD9",
        700: "#0050B3",
        800: "#003A8C",
        900: "#002766"
      }
    }
  }
}
```

Then use it in your components:

```astro
<div class="bg-brand-500 text-white">
  Custom branded section
</div>
```

## Dark Mode Configuration

The portfolio is configured for class-based dark mode:

```javascript
darkMode: "class"
```

This means dark mode is toggled by adding/removing the `dark` class from the `<html>` element. The portfolio's theme toggler handles this automatically.

## Safelist Patterns

The portfolio uses dynamic color classes that need to be safelisted to prevent Tailwind from purging them:

```javascript
safelist: [
  {
    pattern:
      /text-(blue|green|orange|red|yellow|purple|pink|indigo|rose|violet|cyan)-(50|100|200|300|400|500|600|700|800|900)/,
  },
  {
    pattern:
      /bg-(blue|green|orange|red|yellow|purple|pink|indigo|rose|violet|cyan)-(50|100|200|300|400|500|600|700|800|900)/,
  },
]
```

These patterns ensure that technology tag colors (used in projects and experiences) are preserved in the production build.

### Adding New Safelist Patterns

If you add custom dynamic classes, add them to the safelist:

```javascript
safelist: [
  // Existing patterns...
  {
    pattern: /border-(brand)-(400|500|600)/,
  },
  {
    pattern: /shadow-(sm|md|lg|xl)/,
  }
]
```

<Warning>
  Only safelist classes that are generated dynamically (e.g., from JSON data). Static classes in your templates are automatically detected by Tailwind.
</Warning>

## Typography Customization

The portfolio uses Inter variable font. To change the font:

<Steps>
  <Step title="Install your preferred font">
    Using fontsource variable fonts:

    ```bash
    bun add @fontsource-variable/your-font-name
    ```
  </Step>

  <Step title="Import in your layout">
    Replace the Inter import in your main layout file:

    ```javascript
    // Instead of:
    import '@fontsource-variable/inter';
    
    // Use:
    import '@fontsource-variable/roboto';
    ```
  </Step>

  <Step title="Update Tailwind config">
    Add font family configuration:

    ```javascript
    theme: {
      extend: {
        fontFamily: {
          sans: ['Roboto Variable', 'system-ui', 'sans-serif'],
        },
        // ... rest of config
      }
    }
    ```
  </Step>
</Steps>

## Complete Tailwind Config Example

Here's the full `tailwind.config.mjs` structure:

```javascript
/** @type {import('tailwindcss').Config} */
export default {
  content: ["./src/**/*.{astro,html,js,jsx,md,mdx,svelte,ts,tsx,vue}"],
  darkMode: "class",
  theme: {
    extend: {
      colors: {
        gray: {
          // ... gray colors
        },
        dark: {
          // ... dark mode colors
        },
        accent: {
          // ... accent colors
        },
      },
    },
  },
  plugins: [],
  safelist: [
    {
      pattern:
        /text-(blue|green|orange|red|yellow|purple|pink|indigo|rose|violet|cyan)-(50|100|200|300|400|500|600|700|800|900)/,
    },
    {
      pattern:
        /bg-(blue|green|orange|red|yellow|purple|pink|indigo|rose|violet|cyan)-(50|100|200|300|400|500|600|700|800|900)/,
    },
  ],
};
```

## Using Custom Colors in Components

Once you've defined custom colors, use them throughout your Astro components:

```astro
<!-- Background with dark mode variant -->
<section class="bg-gray-50 dark:bg-dark-800">
  <!-- Accent colored heading -->
  <h2 class="text-accent-500">
    Section Title
  </h2>
  
  <!-- Card with dark mode support -->
  <div class="bg-white dark:bg-dark-700 border border-gray-200 dark:border-dark-600">
    <p class="text-gray-900 dark:text-dark-50">
      Card content
    </p>
  </div>
</section>
```

<Tip>
  Use a color palette generator like [Coolors](https://coolors.co) or [Adobe Color](https://color.adobe.com) to create harmonious color schemes with proper contrast ratios.
</Tip>

<Tip>
  Test your color changes in both light and dark modes to ensure proper contrast and readability.
</Tip>

<Warning>
  When changing the accent color, you may need to adjust shades (50-950) to maintain visual consistency across light and dark modes.
</Warning>
