The portfolio uses Tailwind CSS for styling with custom color palettes and dark mode support.

## Tailwind Configuration

The main styling configuration is in `tailwind.config.mjs`:

```js
/** @type {import('tailwindcss').Config} */
export default {
  content: ["./src/**/*.{astro,html,js,jsx,md,mdx,svelte,ts,tsx,vue}"],
  darkMode: "class",
  theme: {
    extend: {
      colors: {
        gray: { /* ... */ },
        dark: { /* ... */ },
        accent: { /* ... */ },
      },
    },
  },
  plugins: [],
  safelist: [ /* ... */ ],
};
```

### Configuration Options

- `content` - File paths to scan for Tailwind classes
- `darkMode: "class"` - Enables class-based dark mode (using `.dark` class on html element)
- `theme.extend.colors` - Custom color palettes
- `safelist` - Dynamic classes that shouldn't be purged

## Dark Mode Implementation

The portfolio uses Tailwind's class-based dark mode strategy:

### How It Works

1. Dark mode is toggled by adding/removing the `dark` class on the `<html>` element
2. Components use `dark:` prefix for dark mode variants
3. The theme preference is stored in localStorage

### Dark Mode Classes

Example of light/dark mode styling:

```html
<div class="bg-white dark:bg-dark-800
            text-gray-900 dark:text-dark-100
            border-gray-200 dark:border-dark-600">
  Adapts to light and dark mode
</div>
```

### Theme Toggle Component

The theme toggler switches between:
- Light mode: Removes `dark` class from `<html>`
- Dark mode: Adds `dark` class to `<html>`

Preference is persisted in localStorage and applied on page load.

## Safelist Configuration

The safelist prevents Tailwind from purging dynamic classes used in content:

```js
safelist: [
  {
    pattern:
      /text-(blue|green|orange|red|yellow|purple|pink|indigo|rose|violet|cyan)-(50|100|200|300|400|500|600|700|800|900)/,
  },
  {
    pattern:
      /bg-(blue|green|orange|red|yellow|purple|pink|indigo|rose|violet|cyan)-(50|100|200|300|400|500|600|700|800|900)/,
  },
]
```

### Why Safelist?

Skills and tags use dynamic colors from locale files:
```json
{
  "name": "JavaScript",
  "color": "green"
}
```

This generates classes like `text-green-500` and `bg-green-100` at runtime. Without safelisting, these would be purged since they don't appear directly in source code.

### Supported Dynamic Colors

The safelist supports these Tailwind color families:
- blue
- green
- orange
- red
- yellow
- purple
- pink
- indigo
- rose
- violet
- cyan

All shades from 50 to 900 are preserved for both text and background utilities.

## Best Practices

### 1. Use Custom Colors Appropriately

- `gray` - Neutral UI elements, borders, text
- `dark` - Dark mode backgrounds and text
- `accent` - Brand elements, CTAs, highlights

### 2. Always Provide Dark Mode Variants

```html
<!-- ✓ Good -->
<div class="bg-white dark:bg-dark-800">

<!-- ✗ Bad -->
<div class="bg-white">
```

### 3. Use Semantic Color Choices

Match color intensity to context:
```html
<!-- Primary text -->
<p class="text-gray-900 dark:text-dark-100">

<!-- Secondary text -->
<p class="text-gray-600 dark:text-dark-300">

<!-- Muted text -->
<p class="text-gray-400 dark:text-dark-400">
```

### 4. Safelist Only What's Dynamic

The current safelist covers skill/tag colors. If you add new dynamic color features, update the safelist patterns.

### 5. Test Both Themes

Always verify components look good in both light and dark modes.

## Customizing Colors

To change the accent color:

1. Update the `accent` scale in `tailwind.config.mjs`
2. Generate a full scale (50-950) for consistency
3. Keep sufficient contrast for accessibility

### Generating Color Scales

Tools to generate Tailwind color scales:
- [Tailwind Color Shades Generator](https://uicolors.app/create)
- [Palette Generator](https://tailwindshades.com/)

## Performance Optimization

The configuration includes:

1. **PurgeCSS**: Unused styles are removed in production
2. **Safelist**: Dynamic classes are preserved
3. **Content Scanning**: Only styles used in specified file types are included

Production builds are optimized automatically.
