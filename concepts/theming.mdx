## Overview

The portfolio features a sophisticated theme system that supports both dark and light modes with a custom color palette. The theme persists across sessions using localStorage and respects user system preferences.

## Dark Mode Configuration

Tailwind CSS is configured to use class-based dark mode in `tailwind.config.mjs`:

```javascript tailwind.config.mjs
export default {
  content: ["./src/**/*.{astro,html,js,jsx,md,mdx,svelte,ts,tsx,vue}"],
  darkMode: "class",
  theme: {
    extend: {
      colors: {
        // Custom color palette
      },
    },
  },
};
```

<Info>
  **Class-based Dark Mode**: The theme is controlled by adding/removing the `dark` class from the root HTML element, enabling instant theme switching without page reloads.
</Info>

## Theme Toggler Component

The theme system is controlled by the `ThemeToggler` React component:

```tsx src/components/ThemeToggler.tsx
import { ICON_SIZE_IN_PX } from "@/config";
import { getTranslation } from "@/i18n";
import type { Locale, Theme } from "@/types";
import { Moon, Sun } from "lucide-react";
import { useEffect, useState } from "react";

interface Props {
  lang: Locale;
}

export const ThemeToggler = ({ lang }: Props) => {
  const [theme, setTheme] = useState<Theme>("dark");
  const [mounted, setMounted] = useState<boolean>(false);

  useEffect(() => {
    setMounted(true);

    const savedTheme = localStorage.getItem("theme") as Theme | null;
    const prefersDark = window.matchMedia(
      "(prefers-color-scheme: dark)"
    ).matches;
    const initialTheme = savedTheme || (prefersDark ? "dark" : "light");

    setTheme(initialTheme);
    applyTheme(initialTheme);
  }, []);

  const applyTheme = (newTheme: Theme) => {
    const html = document.documentElement;

    if (newTheme === "dark") {
      html.classList.add("dark");
    } else {
      html.classList.remove("dark");
    }
    localStorage.setItem("theme", newTheme);
  };

  const toggleTheme = () => {
    const newTheme = theme === "light" ? "dark" : "light";
    setTheme(newTheme);
    applyTheme(newTheme);
  };

  if (!mounted) {
    return <div className="w-10 h-10" />;
  }

  const { t } = getTranslation(lang);

  return (
    <button
      onClick={toggleTheme}
      className="p-2 rounded-lg bg-gray-100 dark:bg-dark-700 cursor-pointer hover:bg-gray-300 transition-colors text-black dark:text-white hover:dark:bg-dark-600"
      type="button"
      title={t("components.theme_toggler.title")}
      aria-label={t("components.theme_toggler.aria-label")}
    >
      {theme === "light" ? (
        <Moon size={ICON_SIZE_IN_PX} />
      ) : (
        <Sun size={ICON_SIZE_IN_PX} />
      )}
    </button>
  );
};
```

## Theme Initialization Flow

<Steps>
  <Step title="Component Mount">
    The component sets `mounted` to true, preventing hydration mismatches.
  </Step>
  
  <Step title="Theme Detection">
    Checks for saved theme in localStorage:
    ```typescript
    const savedTheme = localStorage.getItem("theme") as Theme | null;
    ```
  </Step>
  
  <Step title="System Preference Check">
    If no saved theme exists, checks the user's system preference:
    ```typescript
    const prefersDark = window.matchMedia(
      "(prefers-color-scheme: dark)"
    ).matches;
    ```
  </Step>
  
  <Step title="Theme Application">
    Applies the determined theme (saved > system preference > default dark):
    ```typescript
    const initialTheme = savedTheme || (prefersDark ? "dark" : "light");
    setTheme(initialTheme);
    applyTheme(initialTheme);
    ```
  </Step>
</Steps>

## Theme Persistence

The theme is persisted across sessions using localStorage:

```typescript
const applyTheme = (newTheme: Theme) => {
  const html = document.documentElement;

  if (newTheme === "dark") {
    html.classList.add("dark");
  } else {
    html.classList.remove("dark");

```

<Tip>
  The theme is saved immediately when changed, ensuring the user's preference persists even if they close the browser.
</Tip>

## Preventing Flash of Unstyled Content

### Mounting Guard

The component returns a placeholder during SSR to prevent hydration mismatches:

```tsx
if (!mounted) {
  return <div className="w-10 h-10" />;
}
```

## Dynamic Color Safelist

To support dynamically generated color classes (e.g., skill tags), the Tailwind config includes a safelist:

```javascript tailwind.config.mjs
safelist: [
  {
    pattern:
      /text-(blue|green|orange|red|yellow|purple|pink|indigo|rose|violet|cyan)-(50|100|200|300|400|500|600|700|800|900)/,
  },
  {
    pattern:
      /bg-(blue|green|orange|red|yellow|purple|pink|indigo|rose|violet|cyan)-(50|100|200|300|400|500|600|700|800|900)/,
  },
],
```

<Warning>
  Without the safelist, Tailwind would purge dynamically generated classes that aren't explicitly written in your source files.
</Warning>

## Creating Theme-Aware Components

When building new components, use Tailwind's dark mode variant:

```tsx
// Light mode first, then dark mode override
<div className="bg-white dark:bg-dark-800">
  <h2 className="text-gray-900 dark:text-dark-100">Title</h2>
  <p className="text-gray-600 dark:text-dark-300">Description</p>
</div>
```

## Theme Type Definition

The theme type is defined in TypeScript:

```typescript src/types/index.ts
export type Theme = "light" | "dark";
```

This ensures type safety throughout the application when working with themes.

## System Preference Detection

The theme system respects the user's operating system preference:

```typescript
const prefersDark = window.matchMedia(
  "(prefers-color-scheme: dark)"
).matches;
```

This provides a sensible default when the user visits for the first time, creating a seamless experience that respects their system settings.
